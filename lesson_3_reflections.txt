When would you want to create a remote repository rather than keeping all your work local?

When you want to be able to access your work from different locations. To collaborate with other users and share the repository.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

give you full control. Does not interupt your current work or cause confusion by introducing new changes until you are ready for them.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

Fork clones a repository into another location on GitHub. Clone copies a repository locally a machine. Branches are different versions (commits) within a repository.

What is the benefit of having a copy of the last known state of the remote stored locally?

You can work on the latest version of the remote while offline.
